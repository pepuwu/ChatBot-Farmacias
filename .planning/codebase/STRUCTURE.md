## Directory Purposes

**`src/`:**
- Purpose: All application source code
- Contains: TypeScript modules (transport, handlers, services, AI, config)
- Key files: `index.ts` (entry point), `config.ts` (environment schema)

**`src/ai/`:**
- Purpose: AI response generation and prompt management
- Contains: OpenAI integration, system prompt building, offer generation
- Key files: `openai.ts` (two main functions: `generarRespuesta`, `generarMensajeOferta`)

**`src/whatsapp/`:**
- Purpose: WhatsApp transport layer and business logic
- Contains: Baileys session management, message normalization, two bots (customer + admin)
- Key files:
  - `session-manager.ts`: Multiplex sessions, manage auth state, normalize Baileys messages, route to handlers
  - `customer-bot.ts`: Dispatch to conversation service, detect orders, send responses
  - `admin-bot.ts`: Operator interface for pharmacists (takeover, offers, waitlist)

**`src/telegram/`:**
- Purpose: Super-admin control panel via Telegram
- Contains: Telegraf bot, CRUD operations for pharmacies/pharmacists
- Key files: `bot.ts` (all Telegram commands + handlers)

**`src/services/`:**
- Purpose: Stateless business logic and periodic tasks
- Contains: Conversation state machine, waitlist management, session cleanup
- Key files:
  - `conversation.ts`: Load conversation context, check state, generate AI response, detect orders, manage waitlist
  - `cleanup.ts`: Periodic task to expire inactive conversations

**`prisma/`:**
- Purpose: Database schema definition and migration tracking
- Contains: Prisma schema, auto-generated migration SQL
- Key files:
  - `schema.prisma`: 6 models (Farmacia, Admin, Cliente, Conversacion, Mensaje, ListaEspera)
  - `migrations/`: Timestamped migration files (auto-generated, checked in)

**`dist/`:**
- Purpose: Compiled JavaScript output (generated)
- Contains: Mirrored `src/` structure in `.js` and `.js.map` files
- Committed: No (gitignored)
- Generated: `npm run build` (runs `prisma generate && tsc`)

**`sessions/`:**
- Purpose: Persistent WhatsApp session credentials (Baileys auth state)
- Contains: Per-session subdirectories with `creds.json`, `keys/`, etc.
- Committed: No (gitignored)
- Generated: Created by Baileys on first pairing

**`.planning/codebase/`:**
- Purpose: GSD documentation artifacts
- Contains: ARCHITECTURE.md, STRUCTURE.md, and other analysis documents
- Generated: By `/gsd-map-codebase` command

## Key File Locations

**Entry Points:**
- `src/index.ts`: Application boot sequence, orchestrates all subsystems, graceful shutdown
- `src/server.ts`: HTTP server builder and starter (Fastify)

**Configuration:**
- `src/config.ts`: Environment validation (Zod schema, database URL, tokens, port, paths)
- `.env`: Environment variables (not committed, copy from `.env.example`)
- `tsconfig.json`: TypeScript compiler options (module: esnext, target: ES2022, strict: true)

**Core Logic:**
- `src/services/conversation.ts`: Conversation CRUD, state transitions, history fetch, AI invocation
- `src/ai/openai.ts`: Prompt building, response generation, offer generation
- `src/whatsapp/session-manager.ts`: Baileys multiplex, message routing, QR management

**Database:**
- `prisma/schema.prisma`: Data models (Farmacia, Cliente, Conversacion, Mensaje, ListaEspera, Admin)
- `src/db.ts`: Prisma client singleton (single export: `prisma`)

**Testing:**
- No dedicated test directory; no test files in codebase
- Recommendation: Add `__tests__/` or `*.test.ts` co-located with source

## Naming Conventions

**Files:**
- Pattern: `kebab-case.ts` for modules (e.g., `session-manager.ts`, `customer-bot.ts`)
- Exception: `index.ts` for entry points, `PascalCase` for generated Prisma client

**Directories:**
- Pattern: `lowercase` for feature/layer directories (e.g., `whatsapp/`, `services/`, `ai/`)
- Pattern: `sessions/` stores session-specific subdirectories with pattern `sessionId` (e.g., `admin`, `pharmacy:cuid`)

**Functions:**
- Pattern: `camelCase` with Spanish verb prefixes (e.g., `generarRespuesta`, `obtenerCliente`, `tomarControl`, `handleIncomingMessage`)
- Pattern: `handle*` for event/message handlers
- Pattern: `get*` or `obtener*` for queries
- Pattern: `start*` for async initializers

**Variables/Constants:**
- Pattern: `camelCase` for variables
- Pattern: `UPPER_CASE` for module-level constants (e.g., `ADMIN_SESSION_ID`, `PHARMACY_PREFIX`)
- Example: `const pharmacySessionPrefix = 'pharmacy:'` in customer-bot.ts

**Types:**
- Pattern: `PascalCase` for interfaces (Prisma generates automatically)
- Example: `interface IncomingMessage`, `interface OfertaPendiente`, `interface ControlActivo`

**Exports:**
- Pattern: Named exports for functions/constants; default export avoided
- Pattern: Re-export from `db.ts` and `logger.ts` for singleton access
- Examples:
  - `export const prisma = new PrismaClient();` (src/db.ts)
  - `export const logger = pino({ ... });` (src/logger.ts)
  - `export async function handleIncomingMessage(...) {}` (src/services/conversation.ts)

## Where to Add New Code

**New Customer Interaction Feature (e.g., button menus, multimedia responses):**
- Primary code: `src/services/conversation.ts` (add query/command function) + `src/ai/openai.ts` (update prompt injection if needed)
- Message handlers: `src/whatsapp/customer-bot.ts` (call new service function)
- Database: Update `prisma/schema.prisma` if new model needed
- Tests: Create `src/services/__tests__/conversation.test.ts`

**New Pharmacist Command (e.g., `reportes`, `clientes`):**
- Primary code: `src/whatsapp/admin-bot.ts` (add `handleReportes` function, add command case)
- Business logic: `src/services/conversation.ts` (add query function like `getConversacionesEnEspera`)
- Tests: Add test case to `admin-bot.test.ts`

**New API Endpoint (e.g., external webhook, admin dashboard):**
- HTTP route: `src/server.ts` (add `app.post()`, `app.get()`, etc.)
- Handlers: Extract to `src/api/` directory if multiple endpoints
- Authentication: Add middleware to `server.ts` if required
- Example: Current `/qr/:sessionId` endpoints (lines 23‚Äì87)

**New External Integration (e.g., payment processor, SMS provider):**
- SDK integration: `src/integrations/${provider}/` (e.g., `src/integrations/stripe/`)
- Wrapper functions: `src/integrations/${provider}/client.ts`
- Config: Add env vars to `src/config.ts` (Zod schema)
- Usage: Import from wrapper, call from relevant handler

**New Periodic Task (e.g., send reminders, sync inventory):**
- Task function: `src/services/tasks/${taskName}.ts`
- Starter: `src/index.ts` main() ‚Äî call `start${TaskName}Interval()`
- Shutdown: Add cleanup to shutdown handler (src/index.ts:84‚Äì93)
- Example: Current `startCleanupInterval()` (src/services/cleanup.ts:25‚Äì31)

**Database Schema Change:**
- Update: `prisma/schema.prisma` (add model, field, relation)
- Migrate: `npm run prisma:migrate` (Prisma auto-generates migration)
- Regenerate client: `npm run prisma:generate`
- Commit: Migration file + schema changes

**Telegram Bot New Command:**
- Command handler: `src/telegram/bot.ts` (add `bot.command('name', async (ctx) => { ... })`)
- Authorization: Middleware auto-checks `TELEGRAM_SUPER_ADMIN_ID` (bot.ts:19‚Äì25)
- Database queries: Use `prisma` (import from `src/db.ts`)
- Response: `ctx.reply(message)` or `ctx.replyWithHTML(message)`

## Special Directories

**`sessions/`:**
- Purpose: Persistent WhatsApp session credentials (Baileys auth state)
- Generated: Yes (by Baileys on first QR scan and subsequent connection updates)
- Committed: No (entries are gitignored)
- Cleanup: Delete subdirectory to force re-pairing (user must scan QR again)
- Structure: Each session gets its own subdirectory named by `sessionId` (e.g., `admin/`, `pharmacy:abc123`)
- Files per session: `creds.json` (encrypted credentials), `keys/` (auth keys)
- Note: DO NOT delete `creds.json` while session is connected; will cause 401 loggedOut errors

**`dist/`:**
- Purpose: Compiled JavaScript output
- Generated: `npm run build` (runs TypeScript compiler)
- Committed: No (gitignored)
- Consumed: `npm start` runs `node dist/index.js`
- Watch mode: `npm run dev` uses tsx to compile + watch

**`.claude/` and `.agents/`:**
- Purpose: Claude tooling directories (GSD framework, skills, commands)
- Generated: Yes (by orchestrator)
- Committed: Partially (skills are committed, generated contexts are not)
- Modify: Only `.agents/skills/farmacia-agent/` for project-specific patterns
